# Fido Scanner V2 вЂ” Docs site

Static landing page for the [Fido Scanner V2](https://github.com/The-LukeZ/fido-scanner-v2) Android app. Built with [Astro](https://astro.build) and [Paraglide JS](https://inlang.com/m/gerre34r/library-inlang-paraglideJs) for i18n.

Deployed to GitHub Pages at `https://fidoscan.thelukez.com/`.

## Commands

Run from this directory (`docs/`):

| Command        | Action                                              |
| :------------- | :-------------------------------------------------- |
| `pnpm install` | Install dependencies                                |
| `pnpm dev`     | Start local dev server at `localhost:4321`          |
| `pnpm build`   | Compile messages + build static output to `./dist/` |
| `pnpm preview` | Preview the production build locally                |

## i18n

Locales are defined in `project.inlang/settings.json`. The build produces one static HTML file per locale (e.g. `/en/index.html`, `/de/index.html`). The root `/index.html` redirects to the appropriate locale based on `navigator.language`.

**To add a new locale:**

1. Add it to `locales` in `project.inlang/settings.json` and `astro.config.mjs` (`i18n.locales`)
2. Create `messages/{locale}.json` with all keys from `messages/en.json` translated
3. Run `pnpm build` вЂ” the new locale is picked up automatically everywhere (routes, language switcher, hreflang tags, root redirect)

**To add a new translatable string:**

1. Add the key to `messages/en.json` and all other locale files
2. Use `m.your_key()` in `src/pages/[locale]/index.astro`

## Architecture notes

- `output: "static"` вЂ” fully pre-rendered, no server runtime
- Paraglide's URL strategy is skipped during SSG builds (`isServer = true`), so locale is set explicitly via `overwriteGetLocale(() => assertIsLocale(locale))` in each page's frontmatter using the `[locale]` route param
- Dynamic JS strings (error messages, etc.) are embedded into a `data-strings` attribute at build time and read by the client script at runtime
